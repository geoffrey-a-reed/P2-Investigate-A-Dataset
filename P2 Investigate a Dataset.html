Define Utility Functions¶

In [15]:
# ----------------------------------------------------------------------
# - Define utility functions for later use
# ----------------------------------------------------------------------

def get_labels(s, ascending=True):
    '''Extract an ordered list of unique labels from a pd.Series.'''
    
    return s.sort_values(ascending=ascending).unique().tolist()


def format_float(x, format_str='{0:.2f}'):
    '''Format a numeric value with the given format string.'''
    
    return format_str.format(x)


def format_pct(x, format_str='{0:.2%}'):
    '''Format a percentage with the given format string.'''
    
    return format_str.format(x)


def get_munging_stats(df, print_=True):
    '''Print or return munging statistics for a data frame.'''
    
    munging_stats = pd.concat([df.apply(lambda s: s.isnull()
                                                   .mean())
                                 .apply(format_pct)
                                 .to_frame('Percent Null'),
                               df.apply(lambda s: s.isnull()
                                                   .sum())
                                 .to_frame('Null Count'),
                               df.dtypes
                                 .to_frame('Data Type'),
                               df.apply(lambda s: s.nunique())
                                 .to_frame('Unique Value Count')], 
                              axis=1)
    
    if print_:
        print '-' * 80
        print 'Summary of Data Munging Statistics:'
        print '-' * 80
        print munging_stats
        print '-' * 80, '\n'
    else:
        return munging_stats
    
    
def add_col_on_mapping(df, in_col, out_col, m, int_col=True):
    '''Adds a column to a dataframe based on a dictionary mapping.'''
    
    df[out_col] = df[in_col].apply(lambda k, m=m: m.get(k, np.nan))
    if int_col:
        df[out_col] = pd.Series(df[out_col], dtype='int64')

Load Data¶

Now we're ready to load our data. While we're at it we'll change the column names to lowercase to conform to Python variable naming/attribute access conventions (e.g. my_data.my_column vs. my_data.MyColumn). We'll also replace the arbitrary index pd.read_csv creates with a column from the dataset, but only after we've checked that it's actually a unique index. (Assigning an index automatically with pd.read_csv works, too, but only if you're sure your dataset has a unique index does it work without potential confusion later in the analysis.)

Add Both Integer & Descriptive Columns¶

Depending on how we choose to look at the categorical variables in our dataset it may be useful to have both integer and text labels (i.e. {0,1} vs. {'female', 'male'}). Both conventions have advantages. (For example, for binary data or data that has a natural ordering integers can allow us to use the categories in various analytical functions.) We'll apply that transformation now in order to avoid upsetting the flow of our analysis later:

Explore Latent Variables¶

There are a number of latent variables in the titanic data that we might be able to extract with regular expressions. We'll do that now for the name and cabin columns. While in this case we didn't end up using them further on in our analysis, the habit of extracting useful-looking latent info from text variables could lead us to some insights for other projects.

Represent Continuous Variables as Bins¶

It may be useful to look at continuous variables as binned categories instead. We'll transform age into a binned variable for later use. Another good candidate would be fare, though we won't be using that in our analysis.

In [22]:
# ----------------------------------------------------------------------

range_bins = range(0, int(titanic.age.max()) + 5, 5)

bin_names = [str(range_bin) 
             for range_bin in range_bins]

# Somewhat involved, but boils down to creating 'X to Y' labels
range_labels = [' to '.join(bin_names[idx1:idx2]) 
                for idx1, idx2 in zip(range(0,len(range_bins) - 1), 
                                      range(2,len(range_bins) + 1))]

titanic['age_range'] = pd.cut(titanic.age, bins=range_bins, 
                              labels=range_labels, 
                              include_lowest=False, right=True)

Data Exploration¶

Now that we have our data in a useful form to work with, it's time to explore! As a (somewhat arbitrary) limit on the length and complexity of this project, we're going to restrict ourselves to the variables survived, age, pclass, and sex. We'll be focusing on the following questions:

  • How is passenger age distributed?
  • Does passenger age vary by class and gender?
  • How does passenger survival vary by age, class, and gender?

It's important to remember as we go that any conclusions we reach are inherently tentative. Correlation isn't synonymous with causation. For example, should we discover that older aboard the Titanic survived more often, our conclusion should not be "Increase age causes higher survival rates in early 20th century cruise ship disasters!" We should simply note the correlation between the two variables, taking it as a hint that there might be something to explore with a more rigorous analysis. Essentially, we're able to explain what happened in great detail, but not why it happened.

We can make the following observations:

  • We have more young children and fewer pre-teens than we'd expect given a normal distribution of ages (i.e. the 'bump' between zero and five years)
  • We have more middle aged and elderly passengers than we'd expect given a normal distribution of ages (i.e. the fat tail starting at forty years)
  • Despite these deviations from a normal distribution, mean and median age are close
  • The typical passenger is between twenty and thirty years old

At this point, if we had to characterize the distribution, right skewed would be most appropriate. However, I think the observations above give a better idea of what's going on.

for index, (pclass, ax, cidx_pair, class_annotation) in outer_zipper:

    for sex, cidx, sex_label in inner_zipper:

                     hist_kws={'alpha': 0.25},
                     kde_kws={'shade': False, 'label': sex_label},

    ax.annotate(class_annotation, (2, 0.08))
    ax.set_yticklabels([])
    ax.set_xlim(0, 80)

Here, we can see a clear relationship between passenger class and age. Third class passengers were the youngest, second class passengers next-youngest, and first class passengers the oldest. There are also clear differences between the typical ages of male and female passengers in first and third class (second class men and women appear the most homogenous). There are also some interesting details that might impact further analysis:

  • There are many more young children in second and third class
  • The young children in third class are mostly female
  • There are many more women in their teens and early twenties in first class

We're starting by looking at survival by gender and passenger class. Let's start going through the implications...

  • Women overall are more likely to survive
  • First and second class female passengers are much more likely to survive

Conclusion¶

Summary of Correlation Between Explored Variables:

                Gender    Age Passenger Class Surived?
Gender            1.00   0.09            0.13    -0.54
Age               0.09   1.00           -0.37    -0.08
Passenger Class   0.13  -0.37            1.00    -0.34
Survived?        -0.54  -0.08           -0.34     1.00

This fits nicely with what we've seen so far:

  • Increased survival is associated with decreased gender (i.e. surviving passengers are more likely to be female)
  • Increased survival is associated with decreased passenger class (i.e. closer to 1st, the highest-status class)
  • Increased survival is only weakly associated with passenger age
  • Increased passenger class is weakly associated with increased gender (i.e. lower class passengers are more likely to be male)
  • Increased passenger class is associated with decreased age (i.e. lower class passengers are younger)
  • Increased age is weakly associated with increased gender (i.e. older passengers are slightly more likely to be male)

It's quite tempting to speak of causes when we explore in this fashion:

  • Higher-status passengers received preferential access to lifeboats!
  • Women and children received preferential access to lifeboats!
  • Older passengers with more personal resources were more likely to be able to afford first and second class!

However, such conclusions are premature: they're really better-phrased as questions. Having useful questions to ask is, in fact, one of the best outcomes of data exploration. In this project we've gained a better undertanding of the relationships between gender, age, passenger class, and survival on the Titanic, and we have a place to start for either digging into the historical reasons for these relationships or modeling the relationships between the variables with machine learning.
